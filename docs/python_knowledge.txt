Python Easter Eggs and Philosophy
==================================

import antigravity
-------------------
The 'import antigravity' Easter egg was added to Python 3 by Barry Warsaw in 2008.
When you run this command, Python opens your web browser to https://xkcd.com/353/
which is an XKCD comic titled "Python". In the comic, someone discovers Python and
says "You can fly!" — showing that Python makes complex things simple and fun.
This Easter egg embodies Python's philosophy of making programming enjoyable.
The antigravity module is one of the most famous Python Easter eggs.

import this — The Zen of Python
---------------------------------
Running 'import this' prints the Zen of Python, written by Tim Peters in 2004
as PEP 20. It contains 19 guiding principles for Python design, including:
- Beautiful is better than ugly
- Explicit is better than implicit
- Simple is better than complex
- Complex is better than complicated
- Readability counts
- Errors should never pass silently
- In the face of ambiguity, refuse the temptation to guess
- There should be one obvious way to do it
The source code of the 'this' module is itself an Easter egg — the content
is encoded in ROT-13 cipher inside the module.

Python's Design Philosophy
----------------------------
Python was created by Guido van Rossum and first released in 1991.
The language prioritises readability and simplicity. Its design philosophy
includes: code should be easy to read, one way to do things, batteries
included (rich standard library), and community matters.

Python Error Handling
-----------------------
Python uses try/except blocks for error handling. The philosophy is EAFP:
Easier to Ask Forgiveness than Permission. This means you try an operation
and handle exceptions if they occur, rather than checking preconditions.
Python has many built-in exceptions: ValueError, TypeError, IndexError, etc.

Python Version History
-----------------------
Python 2 was released in 2000 and reached end-of-life in 2020.
Python 3 was released in 2008 and is the current standard.
Python 3.10 added structural pattern matching (match/case statements).
Python 3.11 improved performance by approximately 60 percent over 3.10.
Python 3.12 added even more performance improvements and better f-strings.
